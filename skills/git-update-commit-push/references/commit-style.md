# Commit Style

Prefer short Conventional Commit style titles:

- `feat(scope): add new capability`
- `fix(scope): handle error path`
- `refactor(scope): simplify internal flow`
- `chore(scope): update config/deps`
- `docs(scope): improve usage notes`

Guidelines:

- Keep title under 72 characters when practical.
- Start with a verb (`add`, `fix`, `update`, `remove`, `refactor`).
- Describe what changed, not why it was requested.
- Use scope when it increases clarity, such as `api`, `ui`, `config`, `build`.
